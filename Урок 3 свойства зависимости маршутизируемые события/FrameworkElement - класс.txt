FrameworkElement - класс

Предоставляет относящийся к уровню платформы WPF набор свойств, событий и методов для элементов Windows Presentation Foundation (WPF).
Этот класс представляет реализацию, предоставляемую платформой уровня WPF и которая основана на API-интерфейсах уровня ядра WPF, определенных в классе UIElement.

Иерархия наследования
System.Object 
  System.Windows.Threading.DispatcherObject
    System.Windows.DependencyObject
      System.Windows.Media.Visual
        System.Windows.UIElement
          System.Windows.FrameworkElement
           
Пространство имен:  System.Windows

Использование элемента объекта XAML
<FrameworkElement .../>

Конструкторы

 FrameworkElement	Инициализирует новый экземпляр класса FrameworkElement.

Методы

 AddHandler(RoutedEvent, Delegate)	Добавляет обработчик маршрутизируемого события для указанного маршрутизируемого события, помещая этот обработчик в коллекцию обработчиков текущего элемента. (Унаследовано от UIElement.)
 AddHandler(RoutedEvent, Delegate, Boolean)	Добавляет обработчик маршрутизируемого события для указанного маршрутизируемого события, помещая этот обработчик в коллекцию обработчиков текущего элемента. Чтобы этот обработчик вызывался для маршрутизируемого события, которое уже было отмечено в маршруте события другим элементом как обработанное, укажите для параметра handledEventsToo значение true. (Унаследовано от UIElement.)
 AddLogicalChild	Добавляет указанный объект в логическое дерево данного элемента.

Заметки
FrameworkElement является точкой соединения между классами элементов уровня среды WPF и набора служб презентации UIElement уровня ядра WPF. Дополнительные сведения об этих понятиях см. в разделе Архитектура WPF.
FrameworkElement расширяет UIElement и добавляет следующие возможности:
Определение системы структуры: FrameworkElement предоставляет специальные реализации уровня среды WPF для некоторых методов, определенных как виртуальные члены в UIElement. Наиболее важно, что FrameworkElement запечатывает некоторые переопределения структуры уровня ядра WPF, предоставляя вместо этого эквивалент уровня среды WPF, который должны переопределять производные классы. Например, FrameworkElement запечатывает ArrangeCore, но предоставляет ArrangeOverride. Эти изменения отражают тот факт, что на уровне среды WPF расположена полная система структуры, которая может визуализировать любой производной класс FrameworkElement. На уровне ядра WPF некоторые члены, которые структурируют общее решение структуры на основе WPF, расположены на месте, но фактический механизм системы структуры не определен. Для получения дополнительной информации см. Макет.
Логическое дерево: общая модель программирования WPF часто выражается в виде дерева элементов. Поддержка выражения дерева элементов в виде логического дерева и сопутствующая поддержка определения этого дерева в разметке реализованы на уровне FrameworkElement. Однако обратите внимание, что FrameworkElement произвольно не определяет модель содержимого, оставляя эту ответственность производным классам. Для получения дополнительной информации см. Деревья в WPF.
События времени жизни объекта: это часто полезно знать, при инициализации элемента (вызывается конструктор) или когда элемент сначала загружается в логическом дереве. FrameworkElement определяет несколько событий, связанных с временем жизни объекта, и предоставляющих полезные обработчики для кода операции, которые включают элементы, например, добавление нескольких дочерних элементов. Для получения дополнительной информации см. События времени жизни объекта.
Поддержка привязки данных и динамических ссылок на ресурсы: поддержка привязки данных и ресурсов на уровне свойств реализована классом DependencyProperty и внедрена в систему свойств, однако способность разрешения значения члена, хранящегося в виде Expression (базовая программная конструкция как для привязки данных, так и для динамических ресурсов) реализована в FrameworkElement. Дополнительные сведения см. в разделах Общие сведения о связывании данных и Ресурсы XAML.
Стили: FrameworkElement определяет свойство Style. Однако FrameworkElement еще не определяет поддержку шаблонов и не поддерживает декораторы. Эти функции реализуются классами элементов управления, такими как Control или ContentControl.
Дополнительная поддержка анимации: некоторая поддержка анимации уже определена на уровне ядра WPF, но FrameworkElement расширяет ее реализацией BeginStoryboard и связанных членов.
Как видно из иерархии классов, многие классы WPF являются производными от FrameworkElement, либо непосредственно, либо через промежуточные базовые классы, такие как Panel или Control.
Если предполагается использовать FrameworkElement в качестве базового класса, может потребоваться сначала проверить существующие производные классы. FrameworkElement обеспечивает поддержку нескольких основных сценариев, но также не имеет ряд возможностей, которые предпочтительны для «элемента» в смысле стандартного блока, который используют для создания пользовательский интерфейс в Язык XAML (Extensible Application Markup Language). Например, FrameworkElement не определяет никакой истинной модели содержимого; FrameworkElement как базовый класс не определяет свойство, которое может создавать дочерние элементы Язык XAML. В частности, может понадобиться обратиться к Control и ContentControl.
